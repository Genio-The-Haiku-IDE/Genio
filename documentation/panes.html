The Panes

Genio's main window contains three panes besides the toolbar and the editor view: The Projects pane on the left, the Outline pane on the right, and the Info pane at the bottom. They can be toggled on and off with the first three icons in the toolbar, or from the menu Window|Appearance.

The Projects pane

The Projects pane holds two tabs: the first to manage the files and folders of open projects, the second to manage source control related issues.

Projects

You can have multiple projects open at the same time, but only one can be "active". This is indicated by a bolded project name.
Commands like Project|Build project or a search in the whole project work only on the currently active project. A project can be Set active from the context menu when right-clicking it.

The context menu provides more commands to manage your project, depending on what item it was invoked on. Right-clicking the project name (= top folder), lets you build your project, close it, open its Project settings, switch branches, add new files from templates or open its folder in Tracker, among other things.

A right-click on files or folders inside the project allows you to rename, delete or open a file with its preferred application, etc.

Double-clicking a file opens it in the editor view, if it's a text file. Filetypes unsupported by Genio get opened in their preferred application. For example, a Bookmark file opens in WebPositive and an icon in Icon-O-Matic.

You can drop a folder from a Tracker window onto the pane to add it as another project.

Source control

In this tab you find most often needed actions to interact with your git repository. It also shows the structure of your repository; your local and remote branches, forks and tags.

You switch to a different branch with a double-click, or by choosing from the Current branch menu.

A right-click on a branch opens a context menu to switch, create, rename or delete branches, etc.

With the ⁝ icon at the right above the list, you can execute several git commands like fetch or stash or apply stashed changes.

If there isn't a git repository already, Genio offers you to Init a repository.

The Info pane

The Info pane consists of four tabs: Problems, the Build log, the Console I/O and Search results.

Problems

Clang is continuously analyzing the current file in the editor view and reports any problems it thinks it has found. Sometimes those are false positives and can be ignored. Often it's very helpful, however, to spot typos or missing commas etc.
A right-click on a row offers to fix the issue, if possible.

Build log

When you're building your project, the "Build log" tab pops up, showing the build process.
Warnings and errors are printed in red, showing in which file, on what line and column the error was encountered. Click on it, and you're transported right to it.

The buttons on the right Clear the log or Stop the running process.
The checkboxes above them de/activate stdout and stderr. You can Wrap long lines, and de/activate the Banner which prints the executed command at the start and end of the build process.

Console I/O

If you execute a command from the Git menu, like git status, or choose to Run console program or Run target from the Toolbar, its output will appear in the "Console I/O" tab.

The buttons and checkboxes do the same as described above for the "Build log".

Search results

If you click the icon Find in project in the Find bar, this tab will show all the hits from all files of the currently active project. Double-click to jump to the search result.

To the right is a search panel that is similar to the 'Find bar'. It is in fact synced with the 'Find bar' whenever you do a Find in project there. The additional Project menu let's you select another but the currently active project as a search target.

The Outline pane

The Outline pane shows all symbols – methods, functions, variables, classes, enums, etc. – of the current file. Normally ordered as they appear in the file, you can sort them alphabetically by clicking the icon-button at the top.

Double-clicking an entry jumps to it in the file currently open in the editor.

Right-clicking an entry opens a context menu where Go to symbol does the same as a double-click and Rename symbol… changes the name of that symbol in the whole project.

For this to work correctly in all situations, you need to create a compile database file (compile_commands.json). See Configuring Clangd / LSP on how that can be done very easily.
